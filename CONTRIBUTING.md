# Contributing

Thanks for your interest in `notion-cli`. A few things to know before you open a PR.

## This repo is generated, then patched

`notion-cli` was generated by [CLI Printing Press](https://github.com/mvanhorn/cli-printing-press) and patterned after the [printing-press-library](https://github.com/mvanhorn/printing-press-library). The generated tree is the source of truth for *most* of the code; local edits are tracked in [`.printing-press-patches.json`](./.printing-press-patches.json) so they survive a regeneration.

**Before changing code, ask:**

1. **Is this a generator bug?** If the fix belongs in every CLI the press emits, file it upstream at [`mvanhorn/cli-printing-press`](https://github.com/mvanhorn/cli-printing-press) and link the issue here.
2. **Is this a Notion-specific extension?** Then add it here, and append an entry to `.printing-press-patches.json` with a `summary`, `reason`, `files`, and (if it's a temporary bridge for missing generator capability) `deferred_to_upstream` + `upstream_issue`. See [AGENTS.md](./AGENTS.md) for the exact schema.

## Local dev

First-time setup (per clone):

```bash
make tools          # install golangci-lint v2 (built with your local Go)
make hooks          # enable the pre-push hook (runs `make check` before every push)
```

Everyday commands:

```bash
make build-all      # builds ./bin/notion-pp-cli and ./bin/notion-pp-mcp
make test           # go test ./...
make lint           # golangci-lint run
make check          # everything CI runs: vet + test + lint + build — run this before pushing
```

`make check` mirrors the GitHub CI jobs exactly, so a green `make check` locally means a green CI. The pre-push hook runs it automatically; bypass with `git push --no-verify` only in an emergency.

Smoke test against a real Notion workspace:

```bash
export NOTION_BEARER_AUTH=ntn_...
./bin/notion-pp-cli doctor
./bin/notion-pp-cli notion-search --query "test" --json
```

## PR checklist

- [ ] `go vet ./...` passes
- [ ] `go test ./...` passes
- [ ] `golangci-lint run` passes
- [ ] If the diff touches command surface, `notion-pp-cli --help` reflects it
- [ ] If the diff is a local extension to the generated tree, `.printing-press-patches.json` has a new entry
- [ ] README updated if user-visible behavior changed

## Reporting bugs

Use [GitHub issues](https://github.com/glbinv/notion-cli/issues). Include:

- `notion-pp-cli --version`
- `notion-pp-cli doctor --json` output (redact tokens)
- The exact command you ran and the full output
- Whether it reproduces against the [official Notion MCP](https://github.com/makenotion/notion-mcp-server) (helps us decide if it's a Notion API issue vs. ours)

## Security

Don't open a public issue for security reports. Email the maintainer or use GitHub's private vulnerability reporting.
